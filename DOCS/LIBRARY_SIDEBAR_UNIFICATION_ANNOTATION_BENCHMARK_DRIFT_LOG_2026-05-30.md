# Library Sidebar Unification - Annotation Benchmark Drift Log (2026-05-30)

## Purpose

Capture annotation quality drift signals between waves so readability standards remain stable.

## Drift Signals

- Repeated adjective-heavy annotations without distinct intent
- Missing outcome verbs in one-line descriptions
- Inconsistent terminology between index and status summaries
- Overly long annotations that break scan rhythm

## Logging Rule

When drift appears, record the class of drift and the correction pattern used in the current wave.

## Benefit

Prevents gradual decline in quick-start index scanability as artifact count grows.
