# Library Sidebar Unification Archive Annotation Style Guide (2026-05-29)

Status: Active guidance
Date: 2026-05-29
Scope: Normalize short annotations used when describing closure artifacts in the archive and index surfaces.

## Style Rules

1. Lead with the artifact purpose, not the process used to write it.
2. Keep annotations to one short clause when possible.
3. Use the same noun for repeated concepts such as governance, validation, onboarding, and stewardship.
4. Reserve words like active, draft, and checkpoint for meaningfully different states.
5. Avoid stacking multiple near-synonyms in a single annotation.

## Example Patterns

- Good: "Checkpoint for long-tail index alignment"
- Good: "Guide for interpreting focused-gate validation output"
- Avoid: "Helpful comprehensive guide for many validation-related maintenance scenarios"
