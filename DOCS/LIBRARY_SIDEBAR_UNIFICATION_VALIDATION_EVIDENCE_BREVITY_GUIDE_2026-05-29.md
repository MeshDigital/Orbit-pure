# Library Sidebar Unification Validation Evidence Brevity Guide (2026-05-29)

Status: Active guide
Date: 2026-05-29
Scope: Keep validation evidence concise while preserving reliability signals.

## Brevity Rules

1. Report numeric result first.
2. Name suite family once.
3. Mention warning baseline only when changed.
4. Keep validation block to compact lines.
5. Avoid repeating identical validation paragraphs across docs.

## Preferred Pattern

`Focused gate remained green (42 passed, 0 failed); suite family unchanged; warning baseline unchanged.`
