## Decision Table

| Condition | Decision | Action |
| --- | --- | --- |
| Same purpose, newer artifact clearer | Retire old from quick-start | Keep historical reference only if needed |
| Same purpose, both still used by different roles | Retain both | Clarify role boundary in annotation |
| Overlap causes onboarding confusion | Supersede one | Add supersession cue and update index |
| Artifact still referenced by contract and recap history | De-emphasize only | Avoid deleting without planned migration |

## Principle

Retire overlap from active discovery first; preserve only what supports reliable handoff.
