# Library Sidebar Unification Quick-Start Grouping Comparison (2026-05-29)

Status: Active reference
Date: 2026-05-29
Scope: Compare practical grouping strategies for the active sidebar-unification closure quick-start set.

## Comparison

| Grouping model | Best when | Trade-off |
| --- | --- | --- |
| Chronological | Reviewing the maintenance lane in order | Harder for new maintainers to find a topic quickly |
| Theme-based | Looking for governance, validation, or archive guidance | Requires stronger naming consistency |
| Lifecycle-based | Following artifact creation through stewardship | Older artifacts can appear in multiple phases |
| Role-based | Handing work between maintainers, reviewers, and document stewards | Needs a stable role legend |

## Recommendation

1. Keep the top-level index theme-based for fast scanning.
2. Use lifecycle and role-based grouping only inside supporting notes.
3. Avoid duplicating the same artifact under too many headings.
4. Let the documentation-index contract remain the final discoverability backstop.
