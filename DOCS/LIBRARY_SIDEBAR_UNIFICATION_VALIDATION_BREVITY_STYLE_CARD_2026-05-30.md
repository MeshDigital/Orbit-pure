# Library Sidebar Unification - Validation Brevity Style Card (2026-05-30)

## Purpose

Define the standard style for concise and repeatable validation reporting across recap surfaces.

## Style Rules

- Use one verification block per wave
- Report suite set once, then final totals
- Keep warning baseline mention optional unless changed
- Always include explicit pass statement before planning roll

## Anti-Patterns

- Repeating full command output in recaps
- Mixing verification details into planning bullets
- Omitting test totals after gate execution

## Adoption

Apply this style card to RECENT_CHANGES and both plan summaries in each wave.
